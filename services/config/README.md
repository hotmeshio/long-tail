The `config` module manages workflow configuration stored in PostgreSQL (`lt_config_workflows` and related tables). It is split into four files by concern: `read.ts` for querying individual or all workflow configs, `write.ts` for upserting and deleting configs (transactional), `cache.ts` for loading all configs into an in-memory `Map<string, LTResolvedConfig>` used by the interceptor, and `provider.ts` for fetching completed task data that a workflow declares it consumes. The barrel `index.ts` re-exports everything so existing imports like `import * as configService from '../services/config'` continue to work unchanged.

All SQL statements are externalized in `sql.ts` and imported by name. `upsertWorkflowConfig` runs in a transaction to atomically replace roles, invocation_roles, and lifecycle hooks alongside the workflow row; the `ON CONFLICT (workflow_type) DO UPDATE` pattern ensures idempotent config registration. `listWorkflowConfigs` issues four parallel queries and assembles results in-memory to avoid a complex multi-JOIN. All role references go through `lt_roles` FK constraints, and the `ENSURE_ROLE_EXISTS` upsert guarantees referential integrity without pre-registration.
