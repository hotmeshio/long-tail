Workflow export service — converts HotMesh durable job state into structured, inspectable formats for the dashboard and API consumers. `index.ts` is the public API (barrel); `client.ts` creates HotMesh handles; `direct-query.ts` reconstructs execution history from raw `durable.jobs` rows when the in-memory handle has expired; `enrichment.ts` resolves HotMesh symbol codes and attaches activity/child-workflow inputs; `post-process.ts` injects synthetic timeline events (signal waits, fire-and-forget markers) for visualization; `parsers.ts` contains shared string-extraction and value-parsing helpers used by the other modules.

All SQL is centralized in `sql.ts`. Queries target `durable.jobs` and `durable.jobs_attributes` which are HotMesh's internal tables, not Long Tail's — these tables use `key` as a text primary key with prefix-based LIKE lookups for activity job discovery. `RESOLVE_SYMBOL` is cached in-process since symbol mappings are immutable for a namespace's lifetime. The `JOIN` in `GET_ACTIVITY_INPUTS` leverages the `jobs_attributes.job_id` FK index. Child workflow input lookups use a parameterized `IN(...)` rather than LIKE for exact-match efficiency.
