Task service module for managing LT task records in PostgreSQL. CRUD operations live in `crud.ts`, workflow handle resolution in `resolve.ts`, process-level aggregate queries in `process.ts`, and shared types/constants in `types.ts`. The barrel `index.ts` re-exports everything so existing imports from `services/task` continue to work unchanged.

SQL considerations: `lt_tasks.workflow_id` is indexed for fast lookup; `APPEND_MILESTONES` uses `|| $1::jsonb` for atomic JSONB array append without read-modify-write; `resolveWorkflowHandle` performs a three-tier fallback (lt_tasks -> durable.jobs -> lt_config_workflows) to handle both leaf workflows and containers; `listProcesses` uses `HAVING` on aggregated counts for status filtering since process status is derived from child task states, not stored directly; interval values in process stats are whitelist-validated via `VALID_PERIODS`.
