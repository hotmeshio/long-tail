User service module providing CRUD operations, role management, RBAC authorization helpers, and password authentication for lt_users. Types and constants live in `types.ts`, database CRUD in `crud.ts`, role queries in `roles.ts`, role-based access checks in `rbac.ts`, and password verification in `auth.ts`. The barrel `index.ts` re-exports the public API.

All static SQL is centralized in `sql.ts` and imported as named constants. The `lt_user_roles` join table uses a composite primary key on `(user_id, role)`, which the `UPSERT_USER_ROLE` query leverages with `ON CONFLICT (user_id, role) DO UPDATE SET type = EXCLUDED.type` for atomic role-type promotion. `attachRolesToMany` uses `ANY($1)` for batch role loading to avoid N+1 queries when listing users. The dynamic `UPDATE` built in `updateUser` and the dynamic `SELECT` in `listUsers` remain inline because their SQL is constructed at runtime, but all parameters are positionally bound to prevent injection.
