Workflow interceptor — wraps every registered HotMesh workflow to provide config-driven routing, escalation management, and parent signaling. `index.ts` contains `registerLT()` (one-call setup) and `createLTInterceptor()` (the main factory that routes workflows as containers, LT leaves, or pass-throughs); `activity-interceptor.ts` inspects proxied activity results for milestone events; `state.ts` defines `InterceptorState` and envelope extraction; `escalation.ts` and `completion.ts` handle the two terminal paths (escalation record creation vs. parent signaling + milestone publishing); `context.ts` provides `AsyncLocalStorage` for orchestrator identity propagation; `activities/` contains the proxy activities that bridge the durable sandbox to DB operations (task CRUD, escalation CRUD, config lookup, workflow ops).
