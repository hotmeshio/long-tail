Escalation service module for managing human-in-the-loop escalation records in PostgreSQL. CRUD operations live in `crud.ts`, bulk batch operations in `bulk.ts`, list/stats queries in `queries.ts`, and shared interfaces and constants in `types.ts`. The barrel `index.ts` re-exports everything so existing `import * as escalationService from '../services/escalation'` paths continue to work unchanged.

All static SQL is externalized in `sql.ts`. Notable SQL considerations: the `claimEscalation` CTE captures previous state atomically to detect claim extensions; `assigned_until > NOW()` acts as a soft lock that auto-expires; bulk operations use `ANY($1::uuid[])` for set-based updates; `getEscalationStats` uses `FILTER (WHERE ...)` aggregations to compute multiple status counts in a single table scan; dynamic sort in `listEscalations` is whitelist-validated via `SORTABLE_COLUMNS` to prevent injection.
