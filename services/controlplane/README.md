Control plane service — wraps HotMesh's quorum APIs (`rollCall`, `throttle`) to provide mesh observability and backpressure control for the admin dashboard. `index.ts` exports app discovery, roll call execution, and throttle commands; `sql.ts` contains queries for the `hotmesh_applications` registry table.

Application discovery queries the `hotmesh_applications` table directly (a HotMesh-managed hash table in the public schema). Keys follow the `hmsh:a:{appId}` pattern; rows with a non-null `expiry` in the past are soft-deleted and excluded. For each discovered appId, a read-only `HotMesh.init()` engine is created and cached — this engine connects to the same Postgres quorum channel as the workers without running any workflows itself, enabling `rollCall()` (service discovery ping/pong) and `throttle()` (broadcast rate-limit commands) to reach all mesh members.
