Control plane service — wraps HotMesh's quorum APIs (`rollCall`, `throttle`) to provide mesh observability and backpressure control for the admin dashboard. `index.ts` exports app discovery, roll call execution, and throttle commands; `sql.ts` contains queries for the `hmsh_applications` registry table.

Application discovery queries the `hmsh_applications` relational table for active apps. For each discovered appId, a read-only `HotMesh.init()` engine is created and cached — this engine connects to the same Postgres quorum channel as the workers without running any workflows itself, enabling `rollCall()` (service discovery ping/pong) and `throttle()` (broadcast rate-limit commands) to reach all mesh members.
